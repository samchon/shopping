# `@samchon/shopping-api`
[![npm version](https://img.shields.io/npm/v/@samchon/shopping-api.svg)](https://www.npmjs.com/package/@samchon/shopping-api)

SDK package of the [`@samchon/shopping`](https://github.com/samchon/shopping) monorepo — auto-generated from the backend by [Nestia](https://nestia.io).

Every API endpoint is represented as a typed function with request/response DTOs, so consumers (like `packages/frontend`) call the backend through plain function calls instead of hand-written HTTP requests.

## Structure
```
src/
├── functional/   # generated endpoint functions (one per controller method)
├── structures/   # generated DTO interfaces
└── index.ts      # barrel export
swagger.json      # OpenAPI spec, also generated by Nestia
```

## Commands
```bash
pnpm build          # SDK codegen + tsc + rollup
pnpm build:sdk      # regenerate SDK from backend source (runs nestia in packages/backend)
pnpm build:tsc      # compile TypeScript
pnpm build:rollup   # bundle CJS + ESM
```

## How It Works
The backend's `nestia.config.ts` points its output to this package:

```ts
output: "../api/src",
swagger: { output: "../api/swagger.json" },
```

Running `pnpm build:sdk` in this package triggers `pnpm build:sdk` in `packages/backend`, which runs `nestia all` to regenerate `src/functional/`, `src/structures/`, and `swagger.json`.
